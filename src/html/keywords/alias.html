Alias

An alias is a name that refers to a previously defined type. It can be used to make the code more readable and give values a specific meaning.

You can create an alias with the alias keyword.

Example

alias MyAlias: int;

An alias is in any way compatible with the type it refers to, but not to other aliases of the same type.

Example

alias MyAlias: int; alias MyAlias2: int; let value: MyAlias = 123; let value2: MyAlias2 = 123; value = value2; // Error: MyAlias is not castable to MyAlias2

Public Aliases

Normally an alias exists only in the BBAP file. When it gets compiled to ABAP, the alias is replaced with the underlying type. You can make an alias public with the public keyword. This will make the alias available in the ABAP code.

Example

public alias MyAlias: int;

The alias is now available in the ABAP code. The usage of public aliases is rarely needed, except for code that gets imported from SAP. Like a table.

public alias Vbak: Table<Vbak_Type>;

Methods

You can create methods only for the alias type, only the alias type (or other aliases for this alias) can use the methods.

Example

alias MyAlias: int; extend MyAlias { func timesTwo(): MyAlias { return this * 2; } } let value: MyAlias = 123; let newValue = myAlias.myMethod(); More information at Extend.
